From Leetcode Word Search.

Given a 2D board and a word, find if the word exists in the grid.

The word can be constructed from letters of sequentially adjacent cell, where “adjacent” cells are those horizontally or vertically neighboring. The same letter cell may not be used more than once.

For example,

Given board =

word = "ABCCED", -> returns true,

word = "SEE", -> returns true,

word = "ABCB", -> returns false.

Notes

This one is a kind of DFS(Depth-First Search) problem. For each matched character in board, we search its four directions, we keep doing this process until there is not a right match, then we go back to our previous state. The key of this problem is that we need to construct a two-dimensional array markUsed to mark whether per single character in the board is used, we also need to keep in mind when and where to reset a marked character if it appears a wrong match.

When we find a matched character and it is not the last one of wordArray, we mark it as used, then we determine the characters around it, only if none of its four directions returns true(it means the character is actually a wrong match), the function returns false.

Complexity

Consider the worst situation, which is, every element in board is traversed and the for iteration is executed completely. In this case, the time complexity is O(m^2n^2). And because we used a two-dimensional array markUsed, the space complexity is O(mn).
